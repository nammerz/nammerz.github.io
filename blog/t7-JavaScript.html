I'd like to talk about loops as it is instrumental in both languages to get code to run.

I'll do an example of a “while” loop for both Ruby and JavaScript

In Ruby, a generally written while loop goes like this:

x = gets.chomp.to_i

while x >= 0

puts x

x -= 1

end

Pay attention to how it is written so you can compare it with the JavaScript version. When you run this through terminal, the terminal will wait for a number to be entered; lets use 20 (we don't have to use gets.chomp but I did just for flare. We could have used x = 20). From there, the program will run as follows: While x is greater or equal to 0, which is while 20 is greater or equal to 0, it'll print out 20 and then goes to the next line of code. That line of code will subtract 1 from 20 and run the code again because x is still greater than 0. Since 19 is greater than 0, it will print 19 and then subtract 1 again. The code will run over and over until x is no longer greater than 0.

Now in JavaScript, while loops are done this way:

var x = 20;

while (x >= 0)

{

document.write(x + “
”);

x -= 1;

}

We have to define what x is by starting off with a variable statement “var”. It's required for JS. If you also notice, we have a semicolon at the end of that line. Semicolons serve to separate statements from each other. If you look carefully, I did not use it after the parentheses because that statement is still open. Inside the curly braces “{ }” I used the semicolons to let terminal know that thats the end of each line within the curly braces. I used the same exact variables and objects just to make it easier to understand. Here in JS, you can see that I used document.write. JS does not have a built-in print or display function so we use this. We can use (x); and be done with that but it would show the entire 20 to 0 on one line with no spaces in between. i.e. 201918171615 etc. This is why we add the
so it will print an integer per line. The closing curly braces “}” finishes off the entire statement whereas in Ruby, “end” closes the statement.
